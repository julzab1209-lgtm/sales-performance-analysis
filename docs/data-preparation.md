# Data Preparation

## Overview

Before creating the dashboard, the dataset was prepared and transformed to ensure consistency, accuracy, and usability for analysis in Microsoft Power BI.

The preparation process focused on organizing the data, improving data quality, and creating a reliable model for reporting.

---

## Data Source

The dashboard is based on a structured sales dataset containing information about:

- Products
- Countries
- Sales values
- Units sold
- Dates
- Discounts

---

## Data Cleaning

The dataset was reviewed to ensure data quality before visualization.

Typical preparation steps included:

- Checking for missing values
- Verifying data consistency
- Removing duplicate records (if applicable)
- Validating column data types

---

## Data Transformation

Data was transformed using **Power Query**.

The transformation process included:

- Renaming columns
- Formatting dates
- Adjusting data types
- Organizing tables for reporting
- Preparing data for relationships

---

## Data Modeling

The dashboard uses a structured data model that allows efficient filtering and interaction between visualizations.

The model supports:

- Product analysis
- Country analysis
- Time-based analysis
- Interactive filtering

---

## Measures and Calculations

The report uses calculated measures to support business analysis.

Examples include:

- Net Sales
- Units Sold
- Sales by Product
- Sales by Country

These calculations enable dynamic reporting and interactive analysis within the dashboard.

---

## Outcome

After the preparation process, the dataset was ready for visualization in Power BI.

Well-prepared data improves report accuracy, supports interactive analysis, and provides a solid foundation for business decision-making.
